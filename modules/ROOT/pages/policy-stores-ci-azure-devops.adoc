= Cerbos Hub Azure DevOps Pipelines Integration

Use Azure DevOps Pipelines to upload policies from Azure Repos (or any connected Git provider) into a Cerbos Hub policy store. The pipeline shown here runs on Microsoft-hosted Ubuntu agents and executes the `cerbosctl` container via Docker.

== Prerequisites

* An Azure DevOps organization with a project and repository that contains your policies.
* A Cerbos Hub policy store ID plus API credentials (`CERBOS_HUB_CLIENT_ID` and `CERBOS_HUB_CLIENT_SECRET`).
* Permissions to create pipelines and secret variables in the Azure DevOps project.

== Create `azure-pipelines.yml`

. Add the following YAML file to the root of the repository you will connect to Cerbos Hub:

[source,yaml]
----
# azure-pipelines.yml
trigger:
  branches:
    include:
      - main

pool:
  vmImage: ubuntu-latest

jobs:
- job: UploadCerbosPolicies
  displayName: Upload Cerbos policies
  steps:
    - checkout: self

    - displayName: Upload policies to Cerbos Hub
      script: |
        docker run --rm \
          -e CERBOS_HUB_CLIENT_ID \
          -e CERBOS_HUB_CLIENT_SECRET \
          -v "$(System.DefaultWorkingDirectory)":/app \
          ghcr.io/cerbos/cerbosctl:latest \
          hub store replace-files \
            --store-id="<STORE_ID>" \
            --message="Policy upload from Azure DevOps" \
            /app
            # Optional: append --api-endpoint="https://<region>.cerbos.cloud" if Cerbos provided a custom endpoint
      env:
        CERBOS_HUB_CLIENT_ID: $(CERBOS_HUB_CLIENT_ID)
        CERBOS_HUB_CLIENT_SECRET: $(CERBOS_HUB_CLIENT_SECRET)
----

Replace `<STORE_ID>` with the identifier of your policy store.

== Create the pipeline and variables

. In Azure DevOps, go to *Pipelines* and select *New pipeline*.
. Choose where your code lives (Azure Repos Git, GitHub, Bitbucket, etc.), select the repository that contains the YAML file, and pick *Existing Azure Pipelines YAML file*. Point it to the branch and `azure-pipelines.yml` path.
. When the pipeline editor appears, click *Variables* and create two secrets:
* `CERBOS_HUB_CLIENT_ID` → paste the client ID and enable *Keep this value secret*.
* `CERBOS_HUB_CLIENT_SECRET` → paste the client secret and enable *Keep this value secret*.
. Save the variables and return to the editor.

== Run and monitor the pipeline

. Click *Save and run* to commit the pipeline (or simply *Run* if it already exists).
. Azure DevOps opens the run summary page; select the **Upload Cerbos policies** job to view detailed logs from the Docker command.
. Once the job succeeds, the latest policies from your repository are available inside the Cerbos Hub store. Every subsequent push to the tracked branch will trigger the same pipeline automatically.
